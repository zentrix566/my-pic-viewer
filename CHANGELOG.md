# Changelog

本项目遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)，日期格式为 `YYYY-MM-DD`。

## [1.0.4] - 2026-07-09

### Fixed
- **修复打开白屏问题**：Vite 构建产物使用绝对路径 `/assets/...` 引用静态资源，在 Tauri 自定义协议（`asset://` / `https://tauri.localhost`）下无法正确解析，导致 JS/CSS 加载失败，页面空白。将 Vite `base` 配置改为 `./`（相对路径）后资源可正常加载。

## [1.0.3] - 2026-07-07

### Added
- **检查更新**：工具栏新增 ⏬ 按钮，点击后从 GitHub API 查询最新 Release，有新版时弹出"去下载"确认框，自动打开浏览器跳转下载页面。无需密钥和 CI 改动。
- **右键菜单**：在图片上右键弹出菜单，可快速"复制路径"（📋 复制路径）。
- **图片分类**：工具栏新增📁✅「合适」目录和📁❌「不合适」目录设置按钮，选定目标文件夹后通过绿色「→ ✅ 文件夹名」和红色「→ ❌ 文件夹名」按钮（或快捷键 Z / X）一键将当前图片移动到对应目录并自动翻到下一张。目标目录持久化保存，在状态栏中实时显示。移动前不再二次确认，快速操作。

## [1.0.2] - 2026-07-07

### Changed
- 同目录图片浏览顺序改为 **按文件创建时间倒序（新 → 旧）**，更贴合"按添加顺序浏览"的直觉。
  - 之前：按文件名自然序（`img2.jpg` < `img10.jpg`）。
  - 现在：最近复制/添加进目录的图片会排在前面；创建时间相同则按文件名自然序作为兜底。
  - 拿不到创建时间的平台会自动退回到修改时间。

## [1.0.1] - 2026

### Added
- 画布左右两侧的悬浮翻页箭头，鼠标悬停时显示，便于翻页浏览。

### Changed
- 一些交互与视觉上的细节打磨。

### CI
- GitHub Actions 仅在推送 `v*` tag 时触发构建并发布 Release，避免 `push main` 重复出包。

## [1.0.0] - 2026

首个正式版本。

### Added
- Tauri 2 + Vue 3 + TypeScript 的绿色版图片浏览器骨架。
- 打开单张图片后自动扫描同目录，支持前后翻页、缩略图条。
- 支持格式：jpg / jpeg / png / gif / bmp / webp / ico / tif / tiff。
- 基本操作：缩放、1:1、旋转、复制路径、重命名、另存为、删除到回收站。
- EXIF 信息读取与信息面板展示。
- 关于对话框（作者信息）。
